# Multi-Environment (Phase 2a) Implementation Plan

> **Status: In Progress** — branch `feat/multi-env`, started 2026-07-11

> **For agentic workers:** Read `docs/FEATURE-DEVELOPMENT.md` and the spec. Use `/launchpad-dev` for verification. Commit after each task with the message specified below.

**Goal:** Ambient multi-env context (`X-Launchpad-Environment`), env CRUD, changeset env pin, CLI `env` commands, env-honest reads.

**Architecture:** Middleware resolves env name (default `dev`). Services take env through `resolvePrimary`. Changesets store `environment_id` set on first stage. CLI sticky env + header on all calls. Diff baseline = last release deployed to current env.

**Tech Stack:** Go, chi, cobra, SQLite/Postgres

**Spec:** `docs/superpowers/specs/2026-07-11-multi-env-design.md`

**Branch:** `feat/multi-env`

SQLite migration sketch:

```sql
ALTER TABLE changesets ADD COLUMN environment_id TEXT REFERENCES environments(id);

UPDATE changesets
SET environment_id = (
  SELECT e.id FROM environments e
  WHERE e.project_id = changesets.project_id AND e.name = 'dev'
  LIMIT 1
)
WHERE status = 'open' AND environment_id IS NULL;
```

Postgres: same with UUID types as in 001.

  if v := r.Header.Get("X-Launchpad-Environment"); v != "" {

}
